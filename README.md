RAMoE-Core
Авторегрессионный инференс с O(1) памятью без KV-кэша. Архитектура семантического эха на базе тригонометрического распределения волновых функций и глюонной изоляции контекстов. Метрики CUDA: &lt; 10 мс, VRAM истории: 10.5 КБ.
RAMoE-Core: (Random Access Momentum Energy)

Статус проекта: В процессе разработки. Код является концептуальным прототипом для демонстрации физической модели на реальных тензорах.

RAMoE — это альтернативная архитектура управления контекстом в ИИ, которая полностью ликвидирует KV-кэш на инференсе. Память истории диалога жестко зафиксирована в границах всего 10-20 Килобайт и вообще не растет, сколько бы слов модель ни генерировала дальше.

Вместо того чтобы «зеркалить» и запоминать всю историю реплик (как классический Attention), система работает по принципу «семантического эха» в динамической вязкой среде.

Как это устроена физика движка (Простым языком)

1. Константный контекст O(1): Вся история диалога сжимается на CPU в один легковесный 16-байтовый вектор состояния (Импульс, квантованный Спин, геометрический тангенс направления) и переноситься на GPU единым PCIe-пакетом. Память карты полностью разгружена.
2. Закон семантической пружины (что то похожее на закон Гука): Каждое слово имеет свою Энергию (сложность/массу подлежащего) и Импульс (динамику действия сказуемого). Входящий запрос натягивает семантическую пружину. 
3. Адаптивный коэффициент упругости k: Жесткость пружины подстраивается под тип ввода. Если влетает пустая масса слов без глаголов действия (флуд), пружина становится ультра-жесткой k ~ 0.98 и гасит полуволну за 2–3 слова. Если идет мощный кинетический толчок действием — пружина расслабляется k ~ 0.1, открывая резонансный полет мысли.
4. Сопротивление Ложного Вакуума: Среда обладает инерцией. Текущий фоновый уровень энергии вакуума и эффект «утомления» хорд VRAM динамически ломают и адаптируют коэффициент k прямо в процессе генерации текста.
5. Глюонная изоляция наук (тем): Чтобы модель не смешивала термины, разные дисциплины (математика, физика, когнитивистика, философия) изолированы на уровне CUDA-ядер жесткими масками квантового числа спина. Чужие поля становятся физически прозрачными на GPU без падения скорости вычислений.

 Реальные метрики с моего личного пк (GPU: NVIDIA RTX 3080 Ti ;CPU: i7-9700k)
Потребление VRAM истории: около 12 800 байт (память была статична как на протяжении генерации так и на протяжении нескольких логов как истории диалога, максимальный обьем занятой памяти ~ 20000 байт при тесте около 500 терминов из философии Канта).
Латентность CUDA-ядра: 2.61 мс — 12.44 мс на шаг генерации.

Как запустить демонстрацию

Для запуска не нужны гигабайтные веса и тяжелые зависимости. Требуется только установленный Python и PyTorch с поддержкой CUDA.

1. Запустите пересборку и калибровку высокопрецизионного словаря:
   ```bash
   python vocab_farmer.py
   ```
2. Запустите интерактивный батник или скрипт терминала для общения с моделью (она будет либо философствовать либо нести бред из за недостаточно точного парсера, настраивать нужно скурпулезно):
   ```bash
   python interface_vacuum.py
   ```
